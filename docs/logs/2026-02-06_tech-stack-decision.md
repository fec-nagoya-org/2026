# 技術構成の決定ログ

- 日付: 2026-02-06

## 決定事項

HTML + Tailwind CSS v4 (CLI) による静的サイト構成を採用する。

## 背景

フロントエンドカンファレンス名古屋 2026 の公式Webサイトを制作するにあたり、技術構成を検討した。

## 検討した選択肢

### Astro

- 静的サイト生成に特化しており、コンテンツ駆動のサイトとの相性が良い
- Content Collections によるデータ管理、レイアウトの共通化などの恩恵がある
- ただし、LP1枚構成では共通化するものがなく、セットアップコストに対してリターンが小さい

### Next.js (Static Export)

- React エコシステムの豊富さが強み
- `output: 'export'` で静的サイトとしてビルド可能
- カンファレンスサイト（LP1枚）に対してはオーバースペック

### Vite + React (SPA)

- 軽量でシンプル
- ただしSEO対策にSSR/SSGの仕組みを自前で用意する必要がある
- カンファレンスサイトには向かない

### HTML + Tailwind CSS (CLI) -- 採用

- ビルドツールは Tailwind CLI のみで、構成が最もシンプル
- LP1枚なのでテンプレートエンジンやコンポーネント共通化の必要がない
- 依存関係が最小限で壊れにくい
- 誰でもすぐに編集できる

## 採用理由

- サイト規模がLP1枚であり、フレームワークを導入するメリットが薄い
- ページ分割やレイアウト共通化が不要なため、Astro等のSSGツールは過剰
- Tailwind CLI によるビルドのみで、開発体験と本番パフォーマンスの両立が可能
- 将来ページが増える場合にはAstro等への移行を検討する
